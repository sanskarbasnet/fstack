## What this does

Orchestrates four primitives in sequence:

1. **Coordinate** — `fstack-brain coordinate --topic "<task>"` to surface
   any active overlap or related decisions BEFORE creating the branch.
2. **Branch** — `git checkout -b <slug>` with a sensible name.
3. **Intent** — `fstack-brain intent write` with a draft inferred from the
   task description (user confirms via AskUserQuestion).
4. **Presence** — `fstack-brain heartbeat --status planning` to broadcast
   the new context to other agents.

No new CLI code; this is pure orchestration of existing brain operations.

## When to invoke

- User runs `/parallel <task description>` to start a new feature/branch
- User says "let's start a new task" — agent proactively suggests `/parallel`
  before starting work

## Procedure

### 1. Parse the task

The user types `/parallel add Slack notifications when contractor accepts`.
Extract:
- A 3-6 word title
- A 1-2 sentence body (what + why + wedge)
- A reasonable branch slug: `<agent>/slack-notifications` or
  `<agent>/<slug-of-first-3-keywords>`

### 2. Run /coordinate first

```bash
fstack-brain coordinate --topic "<task>"
```

If `has_overlap=true`, STOP. Surface the overlap to the user. Ask via
AskUserQuestion:
- (a) Coordinate with the other agent (they message Owen/Sanskar)
- (b) Pivot to a complementary angle (suggest one based on the overlap)
- (c) Proceed anyway (they accept the collision risk)

If green, continue.

### 3. Create the branch

```bash
git checkout -b <slug>
```

If branch already exists, ask whether to checkout existing or pick a new slug.

### 4. Draft the intent + confirm

Use AskUserQuestion with the drafted intent:
- title: <auto-generated>
- body: <auto-generated wedge description>

User says yes (or refines). Then write:

```bash
fstack-brain intent write --title "<...>" --body "<...>" \
  --promises "<...>" --not-touching "<...>"
```

### 5. Broadcast presence

```bash
fstack-brain heartbeat --status planning
```

(No --files yet; the agent will fire /touch as it actually picks files.)

### 6. Close

Print a 3-line confirmation:
- Branch created
- Intent locked
- Ready

Then drop into normal coding mode.

## What this skill must NOT do

- Must not create a branch BEFORE running /coordinate. The whole point is
  to surface overlap before creating throwaway state.
- Must not skip the AskUserQuestion confirmation on the intent. The user
  needs one chance to refine before it gets locked in.
- Must not auto-push the new branch. Pushing comes later via /ship.
